Standard Operating Procedures

  • Step 1: Duplicate an exisiting matrix item on the corresponding site portal
    Step 2: Edit product name, description, photos, tax information and SKU (e.g. JMIP1000-MTX) on item.
    Step 3: Edit item price, weight, and SKU on all size variations (If a size is missing, add it in attributes tab).
    Step 4: Publish new item
    Step 5: Sync with NetSuite

Make Scenario Documentation

  • Step 1: This Make scenario runs when a user submits a form by clicking "Create a new Project" in the "Project Team > Client Projects" section of the Knack Live Site. (Here)
    Step 2: First, it creates a record in the "Projects" Knack table and populates it with data from the form (Brand, Store Number, Location Information, Monday Pulse Project Link, Project Description).
    Step 3: Then, using data pulled from the "Projects" Knack table, the scenario creates a unique Google Drive folder for the new project within its respective brand's exisiting folder. (Here)
    Step 4: Next, a collection of nested Google Drive folders are created within the Project folder (Emails, Estimating, Project Files, and Service are the top level folders with more specific folders inside).
    Step 5: Finally, the scenario updates the Project Knack table to include the location of this new folder system.
  • Step 1: This Make scenario runs when a user submits a form by clicking "Create New Order" in the "Warehouse Team" section of the Knack Live Site.
    Step 2: First, it retrieves the new record in the "Pallet-Serial Reports" Knack table and determines whether it is a project or non-project order via the Google Drive folder link.
    Step 3: If the order is in fact a project order, the scenario links the 2 records by updating the data in both tables to reflect their connection.
  • Step 1: This Make scenario runs when a user submits a form by clicking "New Inventory Report" in the "Project Team > Inventory Details" section of the Knack Live Site. (Here)
    Step 2: First, a new row is created in the "Report Request" Knack table and a Google Sheet is created from a template.
    Step 3: Next, the scenario is routed depending on the type of report specified in the initial form (All FKG or Single Reporting Group)
    Step 4: Both routes then access inventory data from NetSuite and use it to populate the Google Sheet Template.
    Step 5: Finally, the scenario appends the new row in Knack with the file ID and location of the new sheet.
